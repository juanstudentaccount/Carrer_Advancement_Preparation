7. What Went Well?

Successes and Best Practices

Weekly check-ins went very well because they kept the project active and ensured I was regularly reflecting on progress. Consistent feedback was essential for staying on track.

Using Trello was also very effective for guiding me through my milestones, and I plan to continue using it for future projects because it helps me visualize tasks and track progress.

8. What Didn't Go Well?

Challenges and Problems

Challenges during my project included keeping up with deadlines while also managing other coursework. Balancing project presentations, labs, and personal assignments at the same time was difficult.

Staying consistent with an organized plan, using tools like Trello, and breaking work into smaller pieces helped me manage this workload and stay on track.

In the future, I would use everything I learned from this project and apply it earlier in the semester. My goal is to continue improving my project management skills so that each new project has a clearer scope, stronger planning, and even more effective execution.

11. Key Takeaways for Future Projects

Top 3â€“5 Actionable Insights:

  • Plan with a realistic scope and timeline. When beginning a project, planning is essential in order to set the scope and set realistic timelines for a single semester.
  • Use a weekly review to adjust the plan early. Revisit status reports every week to check what's on track, what's at risk, and which tasks need to be moved, simplified, or dropped so the overall goal stays realistic.
  • Build quality in from the start, not at the end. Use a checklist for each deliverable (clear writing, correct links, rubric alignment, and visual polish) and review it before submitting, instead of waiting until the final week to fix everything.
